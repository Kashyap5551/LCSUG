Curriculum


  • Year 1 Core

    • EECS1001 - Research Directions in Computing

      • Research Directions in Computing

      This course is about some of the possible research topics in the realm of Computer Science. The topics and lectures vary every year.

      This is definitely the easiest course you'll ever do, and it's pass or fail (no grade received). As long as you attend the classes, do the quizzes at the end of each lecture, and remember to do the assignments - you'll be fine.


      tl;dr - easiest course of your degree. You'll have a good time.


    • EECS1012 - Introduction to Computing: A Net-centric Approach

      • Introduction to Computing: A Net-centric Approach

      This is an introductory course to Computer Science. You'll learn the fundamentals of programming - what variables are, beginner programming logic (conditional logic, loops, program tracing), and the basics of web development.

      You'll be working with HTML, CSS, and JavaScript in this course. If you did any programming on your own or in high school, you'll find this to be a walk in the park. If you didn't, you can still end with a high grade as long as you put in the time and effort; this course is definitely on the easier side of the spectrum.


      tl;dr - if you've programmed before, this'll be easy. If this is your first taste of programming, make sure you learn and understand the concepts.


    • EECS1019 - Discrete Mathematics for Computer Science

      • Discrete Mathematics for Computer Science
      Discrete math is not like your average mathematics such as advanced functions or calculus, with formulas and calculations. This is about the math that is related and necessary for Computer Science. It's how computers think.

      I found the first half of this course to be pretty easy; it was basic logic, logical operators, and set theory. It definitely ramped up a lot when we started doing proofs, induction, counting principles, and more. This is a pretty abstract course and can be difficult to wrap your head around all the concepts, so actually try to pay attention. This is a pretty useful course, I learned a lot and it's really important in being a good developer.

      tl;dr - one of the more challenging first year courses for sure. Make sure you put a lot of time and effort into this.

    • EECS1022 - Programming for Mobile Computing

      • Programming for Mobile Computing
      Definitely a more ramped up and harder course than EECS1012. You'll be learning how to make mobile applications using Java and Android Studio. This would be a good time to mention that Java is the "main" language York University uses; you have another 3-4 courses to take in Java.

      I found this course to be fun - you get to see your code come to fruition and see something real being made from it. That being said, I thought this course was challenging because you're not just learning how to program in a new language, you also have to learn how to use the Android Studio IDE. I actually came into university having never written a single line of code before, so I thought this was hard. I went from an A in EECS1012, to a C in EECS1022.

      tl;dr - if you've programmed before you'll be good. Otherwise, take this course seriously. Learn how to program in Java; we use it a lot.

    • MATH1300 - Differential Calculus with Applications

      • Differential Calculus with Applications
      This is Calculus I, a lot of differentials and a bit of integrals near the end. If you did well in calculus in high school, this will be a breeze for you because almost half of this course is high school content, with the other half being new content that isn't that much more challenging.

      If you struggled with calculus in high school, you'll definitely struggle here too. Read the textbook, pay attention in lectures, and ask for help.

      tl;dr - good at high school calculus? Good. No? Work hard.

    • MATH1310 - Integral Calculus with Applications

      • Integral Calculus with Applications
      This is Calculus II, all about integrals. If you found calculus I to be easy, don't think that this course is anything like it. There is an INSANE ramp up from calculus I to calculus II. I failed this course once before I got my credits in this class, and even then it was tough despite me having already taken it.

      I highly recommend you pay the utmost attention in these lectures and follow the textbook very closely.

      tl;dr - I would say this is one of the hardest first year courses to take. Manage your time well and try not to take this course with other difficult courses.

    • Foundational Science Requirement

      • Foundational Science Requirement
      You need to take 6.00 credits in foundational sciences from a list of courses. Typically this means either two biology, two chemistry, one biology and one chemistry, or one physics course (biology and chemistry courses are 3.00 credits while physics is a 6.00 credit).

      Personally, I went with two chemistry courses since I excelled in these courses in high school. I walked away with an A in both CHEM1000 and CHEM1001, but remember to play to your strength and what you did well in during high school.

      tl;dr - I can't speak for biology or physics, but I went with the chemistry route and found it enjoyable.

    • Electives

      • Electives
      Remember that you need 120 credits to get your degree. The computer science, science, and mathematics requirement don't fill the 120 itself, so you'll be taking a few electives here and there. Generally, I advise against electives in first year unless you know you can handle it. Personally, I was pretty worried for my first year of university and took it easy just to gauge how I'd handle the university life.

      tl;dr - this is up to you and if you feel like you can handle adding an elective to your course load.

      • Software Design - 3.00 credits
      My friends and I joke about this course a lot. We nicknamed this course, "The Gatekeeper" course because it holds people back from graduation.
